# Outputs Folder

This folder stores the generated outputs from the **Dual-Engine 5G Energy Optimisation Framework**. The files in this directory are mainly used for dissertation reporting, model validation, and reproducibility.

## Purpose

The `outputs/` folder contains exported results produced during the evaluation of the dual-engine framework. These outputs support the analysis of:

- baseline network energy consumption,
- surrogate model predictions,
- offline reinforcement learning policy performance,
- oracle validation results,
- statistical comparisons across models,
- figures and tables used in the dissertation.

## Expected Contents

Depending on the experiment run, this folder may include:

```text
outputs/
├── README.md
├── figures/
├── tables/
├── model_metrics/
├── residual_analysis/
├── policy_outputs/
├── oracle_validation/
└── statistical_tests/
```

### 1. Figures

The `figures/` subfolder contains plots exported from the modelling and evaluation pipeline. These may include calibration plots, residual diagnostics, ECDF plots, boxplots, and policy comparison figures.

Examples:

```text
figures/
├── xgb_calibration_plot.png
├── xgb_residual_histogram.png
├── residual_vs_predicted.png
├── policy_energy_comparison.png
└── ecdf_energy_savings.png
```

### 2. Tables

The `tables/` subfolder contains summary tables exported for reporting. These tables may include dataset summaries, model performance metrics, policy comparison results, and final dissertation-ready tables.

Examples:

```text
tables/
├── dataset_summary.csv
├── surrogate_metrics.csv
├── rl_policy_comparison.csv
└── statistical_test_results.csv
```

### 3. Model Metrics

The `model_metrics/` subfolder stores evaluation metrics for the supervised learning and reinforcement learning components of the framework.

Typical metrics include:

- R² score,
- mean absolute error,
- root mean squared error,
- mean absolute percentage error,
- mean energy consumption,
- percentage energy saving.

### 4. Residual Analysis

The `residual_analysis/` subfolder contains outputs used to assess the behaviour of the surrogate model. These files help verify whether the prediction errors are centred around zero and whether the model is suitable for policy evaluation.

Typical contents include:

```text
residual_analysis/
├── residual_summary.csv
├── residual_distribution.png
└── residual_vs_predicted.png
```

### 5. Policy Outputs

The `policy_outputs/` subfolder contains the results generated by the offline reinforcement learning policies. These outputs compare the baseline policy with the optimised policies produced by algorithms such as TD3+BC, IQL, and CQL.

Typical contents include:

```text
policy_outputs/
├── baseline_policy_results.csv
├── td3bc_policy_results.csv
├── iql_policy_results.csv
├── cql_policy_results.csv
└── policy_energy_savings_summary.csv
```

### 6. Oracle Validation

The `oracle_validation/` subfolder contains outputs from the independent validation model used to assess whether the recommended energy-saving actions remain realistic and technically plausible.

Typical contents include:

```text
oracle_validation/
├── oracle_model_metrics.csv
├── oracle_policy_comparison.csv
└── oracle_validation_summary.csv
```

### 7. Statistical Tests

The `statistical_tests/` subfolder contains the statistical evidence used to compare the baseline, surrogate-based optimisation, and offline reinforcement learning policies.

Typical contents include:

```text
statistical_tests/
├── paired_test_results.csv
├── effect_size_results.csv
└── confidence_intervals.csv
```

## Reproducibility Notes

The files in this folder are generated outputs and should not be edited manually. To reproduce the results, rerun the relevant notebook or script from the `notebooks/` or source-code directory.

Recommended workflow:

1. Load and preprocess the source data.
2. Train or load the Engine 1 surrogate model.
3. Evaluate the offline RL policies.
4. Run oracle validation.
5. Export figures, tables, and statistical summaries to this folder.

## Version Control Guidance

Small output files such as `.csv`, `.md`, and selected `.png` figures may be tracked in GitHub for transparency. Large model files, temporary checkpoints, or bulky intermediate outputs should be excluded using `.gitignore`.

Recommended files to keep:

```text
*.csv
*.md
*.png
*.pdf
```

Recommended files to ignore:

```text
*.pkl
*.joblib
*.pt
*.pth
*.ckpt
*.log
__pycache__/
.ipynb_checkpoints/
```

## Academic Use

These outputs were produced as part of the dissertation project:

**Towards Sustainable 5G Networks: A Dual-Engine Framework Using Offline Reinforcement Learning and Surrogate-Based Optimisation**

The outputs support the evaluation of a dual-engine framework combining surrogate-based optimisation and offline reinforcement learning for energy optimisation in 5G radio access networks.

## Notes

This folder is intended to store final or near-final outputs only. Exploratory, temporary, or incomplete files should be kept outside this directory unless they are needed for reproducibility.
